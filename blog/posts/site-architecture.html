Architecture of this Site: A Deep Dive

How I built a performant, modular portfolio using Vanilla JavaScript and Vite, eschewing heavy frameworks for raw speed and control.

Introduction

When building this portfolio, I made a conscious decision to avoid heavy frontend frameworks like React or Vue for the core site. Instead, I chose a stack focused on performance, modularity, and long-term maintainability.

The Stack

  • Vite: For ultra-fast development server and optimized production builds.
  • Vanilla JavaScript (ES Modules): Organized in a modular way (components, managers).
  • CSS Variables: For theming (Dark/Light mode) without CSS-in-JS overhead.
  • GitHub Actions: For CI/CD, including automated PDF generation with Playwright.

Key Structural Decisions

1. Component-Based Architecture without a Framework

I mimicked the component structure of React using native ES Modules. Each major feature (ATS Scorer, Theme Manager) is a self-contained class that initializes itself. 

// Example: ThemeManager.js
export class ThemeManager {
    constructor() {
        this.init();
    }

    init() {
        // ... set up event listeners
    }
}

2. Automated Resume Generation

The most complex part of this project is the dual-mode resume. I use a JSON data source (`data/roles/`) to drive both the website content and the PDF generation.

A GitHub Action spins up the site, uses Playwright to render the "Print" variant, and saves it as a PDF artifact. This ensures my PDF resume is always in sync with my website.

3. PWA & Offline Support

Using a custom Service Worker, the site caches core assets and pages. The `PWAManager` class handles the "Install App" prompt and offline notifications, providing a native-app-like feel on mobile.

Conclusion

This architecture proves that you don't need a heavy framework to build a modern, interactive, and maintainable web application. By sticking to web standards, we get better performance and less dependency drift.
