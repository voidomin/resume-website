Building This Site: A Technical Deep Dive

How I built a performant, accessible portfolio using Vite, Vanilla JS, and GitHub Actions. No frameworks, just solid engineering.

When I set out to rebuild my portfolio, I had one goal: Simplicity. I wanted a site that loaded instantly, scored 100 on Lighthouse, and didn't require me to update npm packages every week.

The Stack

I chose Vite as my build tool. Unlike Webpack, Vite uses native ES modules during development, which means the server starts instantly. For production, it builds heavily optimized assets using Rollup.

CSS Architecture

You won't find Tailwind here. I used CSS Variables for theming (Light/Dark mode) and scoped CSS files for components. This keeps the separation of concern clear and the bundle size tiny. 

:root {
  --primary: #3b82f6;
  --surface: #ffffff;
  --text: #111827;
}

[data-theme="dark"] {
  --surface: #1f2937;
  --text: #f9fafb;
}

Automated Testing

Code without tests is legacy code. I implemented Vitest for unit testing my router and analytics logic, and Playwright for end-to-end testing. Every commit is verified by GitHub Actions before it goes live.

Conclusion

This project proves you don't need a heavy framework to build a modern, interactive web application. Sometimes, vanilla is the best flavor.
